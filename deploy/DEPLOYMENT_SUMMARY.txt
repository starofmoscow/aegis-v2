PROXY SOLUTION (TWO OPTIONS)
─────────────────────────────────────────────────────────────────────────────

OPTION 1: V2Ray SOCKS5 (DEFAULT, RECOMMENDED)
├── Location: System service on VM
├── Port: localhost:10808
├── Access: From app via "socks5://host.docker.internal:10808"
├── Cost: FREE
├── Setup: Automatic (via setup-vm.sh)
├── Supported APIs: All (direct routing)
└── Notes: Full control, low latency

OPTION 2: Cloudflare Worker (FALLBACK, ALTERNATIVE)
├── Location: Cloudflare free tier
├── Cost: FREE (100,000 requests/day included)
├── Setup: Deploy cloudflare-worker-proxy.js to Cloudflare
├── Access: POST https://your-worker.workers.dev/api/{provider}/...
├── Supported APIs:
│   - Anthropic (Claude)
│   - OpenAI
│   - Google Generative AI
│   - xAI (Grok)
│   - DeepSeek
│   - Groq
└── Benefits: Geographic distribution, reliability, easy fallback

RECOMMENDATION:
→ Use V2Ray as primary (auto-configured)
→ Setup Cloudflare Worker as backup/alternative
→ App can intelligently switch between them

TROUBLESHOOTING QUICK FIXES
─────────────────────────────────────────────────────────────────────────────

PROBLEM: App not responding
SOLUTION:
  $ ./deploy.sh logs
  # Check error messages
  $ ./deploy.sh restart

PROBLEM: V2Ray not routing requests
SOLUTION:
  $ ./deploy.sh ssh
  $ nc -zv localhost 10808  # Should respond
  $ journalctl -u v2ray -f  # Check logs
  $ systemctl restart v2ray

PROBLEM: API calls failing
SOLUTION:
  $ ./deploy.sh ssh
  $ docker exec aegis-app printenv | grep API_KEY  # Verify keys present
  $ docker exec aegis-app curl -x socks5://host.docker.internal:10808 https://api.anthropic.com/v1/health

PROBLEM: SSL certificate warning
SOLUTION:
  $ ./deploy.sh ssh
  $ sudo certbot certonly --standalone -d your-domain.com
  # Update nginx.conf with domain paths
  $ docker exec aegis-nginx nginx -s reload

PROBLEM: Out of memory
SOLUTION:
  $ docker stats  # Check usage
  # Edit docker-compose.yml
  # Change: NODE_OPTIONS=--max-old-space-size=1024 (reduce from 2048)
  $ docker compose restart

PROBLEM: Disk full
SOLUTION:
  $ df -h  # Check usage
  $ docker image prune  # Remove unused images
  $ docker system prune  # Deep cleanup
